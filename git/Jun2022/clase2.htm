Segunda sesión curso de control de versiones con git y github:

Temas vistos en la sesión:

  1. Repaso de la sesión anterior,
  2. Que es un commit,
  3. Como hacer commits,
  4. Que es una clave gpg,
  5. como crear una clave gpg,
  6. Como configurar la firma de commits y tags con la clave gpg creada,
  7. Como ver el historial de commits,
  8. Como cambiar a un commit con su identificador,
  9. Como ver información detallada de un commit,
  10. Que son las tags,
  11. como asignar un tag a un commit,

Comandos utilizados en clase.

  • git add .
  • git add "lista de tareas.txt"
  • git commit -m "mensaje que explica el commit"
  • gpg --full-generate-key
  • gpg --list-secret-keys --keyid-format=long
  • gpg --list-secret-keys --keyid-format=long > miGPG.txt
  • git config --global user.signingkey identificador
  • git config --global commit.gpgsign true
  • git config --global tag.gpgSign true
  • git log
  • git log > log.txt
  • git show
  • git tag -m "descripción del tag" nombre
  • git checkout
  • git checkout master

Tarea:

crear un repositorio con tu nombre:

crear dentro un archivo de texto con tu nombre

En el archivo rellenar las siguientes secciones con tu información, real o ficticia.

  • información general,
  • Educación
  • Profesión
  • Información laboral
  • hobbies

Para cada sección crear un commit.

En el mismo repositorio crear 4 archivos más con sus intereses como música, películas, libros, juegos, equipos de football, etc.

En cada archivo describir que es lo que más les gusta, y un mínimo de tres ejemplos de porqué les gusta. Hacer un commit al momento de crear cada archivo, y otro commit cuando hayan completado la información.

Asignar un tag principio al commit raíz, y un tag con el nombre para cada archivo completado, ejemplo: general, música, libros.

Por último, navegar al tag en el que solo está el primer archivo creado.

Comprimir toda la carpeta en un archivo rar o zip. y enviarla al correo electrónico contacto@josiasvasquez.com
